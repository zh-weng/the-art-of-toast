# 敬酒模拟器

双人面对面对战的物理模拟游戏 —— 谁的杯子先干，谁就输了。

## 架构

- **物理引擎**: [Matter.js](https://brm.io/matter-js/) 处理液体粒子和杯子的碰撞、重力
- **渲染**: 自定义 WebGL metaball 着色器（`liquid-renderer.js`）在 GPU 上生成液体融合效果，替代了旧的 SVG gooey 滤镜。Safari 中 WebGL 画布合成走 CPU 路径，因此自动降为半分辨率渲染以保证帧率

## 难度选择

在游戏开始前可以选择难度：
- **简单**：杯壁较直，液体较多（120粒）
- **普通**：杯壁微斜，液体适中（100粒）
- **困难**：杯壁大幅外斜，液体较少（80粒）

## 控制

### 玩家1（左侧杯子）

| 按键 | 动作 |
|------|------|
| `W` | 向上移动 |
| `S` | 向下移动 |
| `A` | 向左移动 |
| `D` | 向右移动 |
| `Q` | 逆时针旋转 |
| `E` | 顺时针旋转 |

### 玩家2（右侧杯子）

| 按键 | 动作 |
|------|------|
| `O` | 向上移动 |
| `L` | 向下移动 |
| `K` | 向左移动 |
| `;` | 向右移动 |
| `I` | 逆时针旋转 |
| `P` | 顺时针旋转 |

移动端支持触摸虚拟按键（自动检测竖屏手持对战模式）。

## 胜负判定

- **液体全部溢出** → 对方胜利
- **两杯相撞时** → 杯口更低的一方胜利（杯口接触判断）

## 游戏特性

- WebGL metaball 着色器液体渲染（GPU 加速，Safari 自动降分辨率）
- 60Hz/120Hz 自适应移动速度（统一帧率体验）
- 精确的碰撞检测（子部件级别边界检查）
- 移动端竖屏面对面布局
- 粒子防重叠物理优化
